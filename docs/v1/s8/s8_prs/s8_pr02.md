# PR-02: YouTube Media Pane + Embed-Safe Transcript UX

## Goal

Ship the end-user video transcript experience in one media pane with embed-safe YouTube playback, deterministic transcript seek, and capability-safe fallback behavior.

## Builds On

PR-01 (YouTube Identity + Ingest + Transcript Contract) merged.

## Acceptance

- transcript-ready `video` media renders in-app playback via approved YouTube embed origins (iframe allowlist) rather than direct file-style browser video playback.
- frontend consumes the typed playback contract from API for video playback behavior (provider-derived watch/embed/source metadata) and does not parse raw URLs client-side to construct embeds.
- transcript-ready videos present co-located playback and transcript interaction in one media view, reusing existing transcript highlight, annotation, and quote-to-chat behavior.
- clicking a transcript segment seeks embedded playback deterministically to that segment’s `t_start_ms` target while preserving stable transcript selection/focus behavior.
- video transcript highlight quote-to-chat preserves transcript metadata context, including timestamp and speaker label when available.
- if in-app playback fails for a transcript-ready video, transcript reading/highlighting/quote-to-chat remain usable and an explicit source fallback action is shown.
- playback-only videos (`processing_status=failed` + `last_error_code=E_TRANSCRIPT_UNAVAILABLE`) show explicit transcript-unavailable state and keep transcript-dependent actions disabled.
- frontend security posture remains constitution-compliant: YouTube-only embed allowlist is enforced, middleware/unit coverage asserts exact `frame-src` allowlist values (`https://www.youtube.com` and `https://www.youtube-nocookie.com`, no wildcard), and document rendering remains iframe-free.
- frontend regression coverage is added for embed rendering, transcript click-to-seek, playback failure fallback, and transcript-unavailable gating; existing transcript media regressions remain green.

## Non-Goals

- no advanced player controls (speed, chapters, playlists, fullscreen orchestration beyond basic embed behavior).
- no semantic/vector ranking or transcript search-contract changes.
- no non-YouTube provider support or YouTube channel subscription features.

## Key Decisions (if any beyond L2)

**Playback Contract Boundary**: media pane behavior is driven by server-derived typed playback metadata; client-side URL-shape inference is not part of the contract.

**Kind-Specific Player Surface**: `video` adopts embed playback while `podcast_episode` keeps existing external-audio behavior, preventing cross-kind regressions in shared transcript UI.

**Failure Handling Posture**: player runtime failures are treated as session-level playback fallback events and must not downgrade transcript-ready capability state.
