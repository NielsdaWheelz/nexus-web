# PR-01: YouTube Identity + Ingest + Transcript Contract

## Goal

Deliver a production-safe backend vertical for YouTube URL ingestion with global idempotency, transcript persistence, typed playback metadata, and explicit playback-only fallback semantics.

## Builds On

Slice 7 merged state (transcript fragments, capability gating, playback-only error semantics, default-library closure).

## Acceptance

- `POST /media/from_url` classifies YouTube URL variants as `media.kind=video`, resolves one canonical provider identity, and creates-or-reuses one shared media row attached to the requester’s default library.
- `POST /media/from_url` exposes explicit idempotency outcome (`created` vs `reused`) for the canonical video identity, with any legacy `duplicate` signaling treated as compatibility-only.
- repeated ingest requests across users converge on that same video media row and reuse existing transcript artifacts/work instead of duplicating rows or transcription attempts.
- video playback metadata is provider-derived and typed from canonical YouTube identity (including provider video ID and canonical watch/embed targets), with compatibility shims treated as transitional only.
- transcript-success ingestion persists canonicalized transcript segments as fragments with deterministic ordering invariants and transitions media to `ready_for_reading` with playable watch-source metadata.
- transcript-unavailable ingestion sets `processing_status=failed` with `last_error_code=E_TRANSCRIPT_UNAVAILABLE`, preserves playback capability, and disables transcript-dependent capabilities (`can_read/can_highlight/can_quote/can_search=false`).
- transcript feasibility is delivered as reproducible artifacts (fixture-backed checks + dated probe report) covering representative YouTube failure modes with measured success/failure rates and verified playback-only fallback behavior.
- library search uses a shared transcript-media searchability predicate so transcript-unavailable items are excluded consistently for both `video` and `podcast_episode`.
- visibility, processing-state, media detail/list, and media/search regression suites remain green with video-specific coverage added.

## Non-Goals

- no frontend YouTube embed rendering or transcript click-to-seek UX in this PR.
- no non-YouTube video providers or YouTube channel subscriptions.
- no advanced player controls or transcript editing/correction tooling.
- no semantic/vector ranking redesign beyond capability-consistent transcript-media filtering.

## Key Decisions (if any beyond L2)

**Canonical Identity Anchor**: video identity and idempotency are keyed by provider identity (YouTube video ID + canonical provider metadata), not by raw URL shape.

**Reuse Before New Work**: ingestion must attach existing global video rows/artifacts first and only create new transcript work when no canonical winner exists.

**Shared Searchability Contract**: transcript-media inclusion in search is driven by one shared predicate aligned with capability posture, preventing `video`/`podcast_episode` behavior drift.

**Idempotency Outcome Contract**: PR-01 makes `created` vs `reused` explicit in ingest responses; legacy `duplicate` fields are transitional and must not remain the long-term source of truth.
