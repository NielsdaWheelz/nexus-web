# PR-10: Podcast Test Coverage Hardening

## Goal

Close the critical test gaps across frontend and backend for the podcast feature set (slices 7 PR-01 through PR-04), ensuring production-grade confidence in subscription lifecycle, transcription pipeline, quota enforcement, semantic search, player integration, and BFF proxy behavior.

## Builds On

PR-04 (Active Subscription Polling) merged. All core podcast backend and frontend code exists but has significant untested paths.

## Problem

The podcast implementation has solid architecture but thin test coverage in several critical areas:

### Backend Gaps

1. **Transcript admission** — the `request_podcast_transcript_for_viewer` flow (quota check → reservation → enqueue) has no dedicated test coverage. This is the highest-risk untested path: a bug here could silently over-spend quota or silently fail to enqueue work.
2. **Transcript forecasting** — the batch dry-run forecast endpoint (`POST /media/transcript/forecasts`) is untested. Users see forecast data before requesting transcription — wrong forecasts erode trust.
3. **Deepgram integration** — no mock/fixture tests for the Deepgram transcription path. The `run_podcast_transcription_now` function calls an external API; we need fixture-based tests verifying segment parsing, diarization fallback, error handling, and timeout behavior.
4. **Subscription sync full cycle** — the end-to-end flow from `subscribe_to_podcast` → `run_podcast_subscription_sync_now` → episode creation → library attachment → transcript job enqueue is not tested as a single flow. Individual pieces are tested, but integration across the chain is missing.
5. **Quota edge cases** — no tests for: exactly-at-limit admission, concurrent reservation races, reservation refund on enqueue failure, daily reset behavior, zero-duration episodes.
6. **Semantic repair workflows** — `repair_podcast_transcript_semantic_index_now` is untested. The migration (0026) backfills embeddings, but the repair path for stale/missing chunks after model changes has no test coverage.
7. **PodcastIndex provider errors** — no tests for: API timeout, rate limit (429 + Retry-After), malformed response, empty results, partial results.
8. **Unsubscribe retention modes** — modes 1/2/3 exist but only the happy path is tested. Edge cases: unsubscribe while sync is running, unsubscribe + resubscribe, mode 2 with shared libraries.
9. **Internal plan route** — `PUT /internal/podcasts/users/{user_id}/plan` authorization enforcement (admin roles, user IDs, emails) is not exercised in tests.
10. **Stale-running reconciliation** — `reconcile_stale_ingest_media` touches podcast media but its interaction with transcript state is untested.

### Frontend Gaps

1. **Global player** — no tests for play/pause, track switching, seek, audio element binding, or the `GlobalPlayerProvider` context. This is core shared infrastructure.
2. **Transcript polling** — `transcriptPolling.ts` and `transcriptPolling.test.tsx` exist as new files but may be incomplete. The polling hook (`useTranscriptProvisioningPoll`) needs tests for: start polling, stop on terminal state, error handling, interval behavior.
3. **Search page** — `search/page.test.tsx` exists as a new file but may be incomplete. The search results adapter for transcript chunks (`resultRowAdapter.ts`) needs coverage for the flat/nested response normalization.
4. **Episode interaction flows** — no tests for: clicking play on an episode row, add-to-library/remove-from-library actions, transcript request from episode list, forecast display.
5. **Subscription management flows** — `podcasts-flows.test.tsx` exists but needs coverage for: unsubscribe mode selection, sync refresh trigger, plan/quota display.
6. **TranscriptMediaPane** — tests exist but need coverage for: all transcript states (not_requested, queued, running, ready, partial, failed_provider, failed_quota, unavailable), budget forecast display, retry action.
7. **BFF proxy routes** — no tests for the Next.js API routes that proxy to FastAPI. These routes do auth forwarding and should verify correct header propagation, error passthrough, and content-type handling.

## Scope

### Backend Tests

All tests should follow the testing standards in `docs/sdlc/testing_standards.md` — behavior-focused, no implementation mocking of internal boundaries, real stack where possible.

#### Transcript Admission Tests (`python/tests/test_transcript_admission.py`)
- request transcript for a valid podcast episode → job created, quota reserved, state transitions to `queued`.
- request transcript with `dry_run=true` → forecast returned, no job created, no quota change.
- request transcript when quota exhausted → rejected with `failed_quota`, no job created.
- request transcript for exactly-at-limit episode (remaining_minutes == required_minutes) → admitted.
- request transcript for episode with `duration_seconds = 0` or null → handled gracefully (0 minutes required, or rejected with validation error).
- request transcript when episode already has `transcript_state = ready` → idempotent, no new job.
- request transcript when episode has `transcript_state = queued` or `running` → idempotent, no duplicate job.
- request transcript for non-podcast media → rejected.
- concurrent transcript requests for the same episode → only one job created (idempotency).

#### Transcript Forecast Tests (`python/tests/test_transcript_forecast.py`)
- batch forecast for 3 episodes → returns per-episode forecast with correct remaining_minutes accounting (each forecast decrements the remaining pool).
- forecast for episodes already transcribed → `required_minutes = 0`, `fits_budget = true`.
- forecast when quota exhausted → `fits_budget = false` for episodes that don't fit.

#### Deepgram Integration Tests (`python/tests/test_deepgram_transcription.py`)
- successful diarized transcription → segments with speaker labels persisted.
- diarization failure → fallback to non-diarized → segments without speaker labels persisted.
- both attempts fail → `failed_provider` state, error code persisted.
- timeout → `E_TRANSCRIPTION_TIMEOUT` error code.
- malformed Deepgram response → graceful failure, not crash.
- segment canonicalization: NFC normalization, whitespace collapse, timing validation.

#### Subscription Sync Integration Tests (`python/tests/test_subscription_sync_integration.py`)
- subscribe → sync → episodes created → library attached → transcript jobs enqueued for episodes with audio URLs.
- second user subscribes to same podcast → no duplicate episodes, no duplicate transcript jobs, library attachment for second user.
- sync with no new episodes → idempotent, no new rows.
- sync with provider error → `sync_status = failed`, error_code set, no partial episode state.
- sync with rate limit (429) → retry logic fires, eventual success or explicit failure.

#### Quota Edge Cases (`python/tests/test_quota_edge_cases.py`)
- exactly-at-limit: 60 minutes remaining, 60-minute episode → admitted.
- one-over-limit: 59 minutes remaining, 60-minute episode → rejected.
- reservation refund on enqueue failure → `minutes_reserved` decremented.
- daily reset: usage on day 1 does not affect day 2 admission.
- concurrent reservations: two simultaneous requests for different episodes → both accounted correctly (no double-spend).

#### Semantic Repair Tests (`python/tests/test_semantic_repair.py`)
- transcript with no chunks → repair creates chunks + embeddings, sets `semantic_status = ready`.
- transcript with stale embeddings (wrong model) → repair regenerates, updates model field.
- repair with OpenAI embedding timeout → `semantic_status = failed`, no partial chunks persisted.
- repair for media with no transcript → no-op or explicit skip.

#### PodcastIndex Provider Tests (`python/tests/test_podcast_index_provider.py`)
- successful search → parsed results returned.
- 429 rate limit with Retry-After → retried after delay, eventual success.
- 500 server error → retried, eventual failure after max attempts.
- timeout → explicit error, not hang.
- malformed JSON response → graceful error.
- empty results → empty list, not error.

#### Unsubscribe Edge Cases (`python/tests/test_unsubscribe_edge_cases.py`)
- mode 1 (keep episodes) → subscription inactive, episodes remain in libraries.
- mode 2 (remove from default library) → episodes removed from default, kept in non-default.
- mode 3 (remove from default + single-member libraries) → episodes removed from default and single-member, kept in shared libraries.
- unsubscribe while sync is running → sync completes but no new episodes added post-unsubscribe.
- unsubscribe + resubscribe → new subscription, fresh sync.

### Frontend Tests

All tests should follow the testing standards — behavior-focused, testing user-visible outcomes.

#### Global Player Tests (`apps/web/src/lib/player/globalPlayer.test.tsx`)
- `setTrack()` updates current track and renders title.
- `setTrack({ autoplay: true })` starts playback.
- `setTrack({ seek_seconds: 120 })` seeks audio element to 120s.
- `play()` / `pause()` toggle `isPlaying` state.
- `seekToMs(30000)` sets audio currentTime.
- `clearTrack()` resets to null state.
- track switching preserves provider binding.

#### Transcript Polling Tests (`apps/web/src/app/(authenticated)/media/[id]/transcriptPolling.test.tsx`)
- polling starts when `transcript_state` is `queued` or `running`.
- polling stops when `transcript_state` reaches `ready`, `partial`, `failed_provider`, `failed_quota`, `unavailable`.
- polling refetches on configured interval (3 seconds).
- error during poll does not crash; retries on next interval.

#### Search Results Adapter Tests (`apps/web/src/lib/search/resultRowAdapter.test.ts`)
- flat response shape → correctly adapted rows.
- nested response shape → correctly adapted rows.
- `transcript_chunk` result type → includes `t_start_ms`, `t_end_ms`, source metadata.
- missing/null fields → graceful defaults, not crashes.

#### TranscriptMediaPane State Tests (`apps/web/src/app/(authenticated)/media/[id]/TranscriptMediaPane.test.tsx`)
- `not_requested` state → shows CTA with budget forecast.
- `queued` state → shows provisioning status.
- `running` state → shows provisioning status + polling active.
- `ready` state → shows transcript segments.
- `partial` state → shows transcript + coverage warning.
- `failed_provider` state → shows retry option.
- `failed_quota` state → shows quota exceeded message.
- `unavailable` state → shows unavailable message.
- clicking a segment seeks player to `t_start_ms`.
- budget forecast renders `requiredMinutes`, `remainingMinutes`, `fitsBudget` correctly.
- forecast with `fitsBudget = false` disables the request button.

#### Podcast Flows Tests (`apps/web/src/app/(authenticated)/podcasts/podcasts-flows.test.tsx`)
- discover → subscribe → subscription appears in list.
- unsubscribe mode selection dialog renders 3 options.
- sync refresh button triggers POST and shows updated status.
- plan/quota snapshot displays tier, used, remaining correctly.

## Acceptance

- all tests listed above pass.
- no test mocks internal service boundaries (only external boundaries: Deepgram, PodcastIndex, OpenAI).
- quota edge cases (at-limit, over-limit, concurrent, refund) are explicitly tested and passing.
- transcript admission idempotency is proven by test (duplicate request = no duplicate job).
- Deepgram diarization fallback is tested with fixture responses.
- subscription sync integration test covers the full chain from subscribe to transcript job enqueue.
- frontend tests cover all 8 transcript states in TranscriptMediaPane.
- global player context tests prove track switching, seek, play/pause without a real audio element (mock HTMLAudioElement).
- existing test suites remain green (no regressions).
- test file organization follows existing patterns in `python/tests/` and `apps/web/src/`.

## Non-Goals

- no E2E tests (browser automation) in this PR — unit and integration only.
- no performance/load tests.
- no tests for PRs 05-09 features (those PRs include their own tests).
- no refactoring of existing tests unless they block new test additions.

## Key Decisions

**Separate test files per concern**: rather than stuffing all new tests into `test_podcasts.py`, create focused test files (`test_transcript_admission.py`, `test_quota_edge_cases.py`, etc.). This matches the testing standards' preference for organized, discoverable test suites.

**Fixture-based external mocking**: Deepgram and PodcastIndex tests use fixture files (JSON response snapshots) rather than inline dicts. Fixtures are easier to update when provider APIs change and serve as documentation of expected response shapes.

**Frontend tests use React Testing Library**: all component tests render with RTL and assert on user-visible outcomes (text, buttons, state changes), not internal component state or implementation details.

## Risks

- some backend tests may require database fixtures that are complex to set up (user + library + media + podcast + subscription + episode chain). Consider a test factory/helper to reduce setup boilerplate.
- concurrent quota tests may be flaky if they depend on transaction isolation timing. Use explicit serializable transactions or deterministic sequencing.
- Deepgram response fixtures may drift from real API responses over time. Pin fixture versions and note the API version they were captured from.
