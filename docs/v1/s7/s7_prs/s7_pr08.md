# PR-08: OPML Import/Export

## Goal

Allow users to bulk import podcast subscriptions from other podcast apps via OPML files and export their current subscriptions as OPML for portability.

## Builds On

PR-04 (Active Subscription Polling) merged. Subscription CRUD and sync pipeline are operational.

## Problem

Users switching from Podcast Addict, Apple Podcasts, Google Podcasts, or any other app expect to bring their subscriptions. Manually re-subscribing to 50+ podcasts one at a time is a dealbreaker for adoption. OPML is the universal interchange format for podcast subscriptions — every major podcast app supports it.

## Background: OPML Format

OPML (Outline Processor Markup Language) is an XML format. Podcast OPML files contain `<outline>` elements with podcast metadata:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<opml version="2.0">
  <head>
    <title>My Podcasts</title>
    <dateCreated>Thu, 20 Mar 2026 12:00:00 GMT</dateCreated>
  </head>
  <body>
    <outline text="Podcasts">
      <outline type="rss" text="Podcast Title" xmlUrl="https://feed.example.com/rss"
               htmlUrl="https://example.com" />
      <outline type="rss" text="Another Podcast" xmlUrl="https://other.example.com/feed.xml" />
    </outline>
  </body>
</opml>
```

Key fields: `xmlUrl` (RSS feed URL — required, this is the identity), `text` (title), `htmlUrl` (website). The `type="rss"` attribute identifies podcast outlines.

## Scope

### Backend

- **New endpoint: `POST /podcasts/import/opml`** — accepts multipart file upload of an OPML file.
  - Parses XML, extracts all `<outline type="rss">` elements.
  - For each feed URL:
    1. Normalize the URL (strip trailing slashes, normalize scheme).
    2. Look up existing `podcasts` row by `feed_url`. If found, reuse it.
    3. If not found, attempt PodcastIndex discovery by feed URL to get provider metadata. If PodcastIndex doesn't know it, create a minimal `podcasts` row from the OPML metadata (title from `text`, feed_url from `xmlUrl`, website_url from `htmlUrl`).
    4. Create subscription if not already subscribed. Skip if already subscribed.
    5. Enqueue sync job for each new subscription.
  - Returns a summary: `{ total: N, imported: N, skipped_already_subscribed: N, skipped_invalid: N, errors: [{ feed_url, error }] }`.
  - Limit: max 200 outlines per import. Reject files with more.
  - Runs synchronously for the parsing/subscription-creation phase; sync jobs run async.
- **New endpoint: `GET /podcasts/export/opml`** — returns an OPML XML file of all active subscriptions.
  - Content-Type: `application/xml`.
  - Content-Disposition: `attachment; filename="nexus-podcasts.opml"`.
  - Includes all `status='active'` subscriptions with `xmlUrl` (feed_url), `text` (title), `htmlUrl` (website_url).
- **Input validation:**
  - Reject non-XML files (check Content-Type and attempt XML parse).
  - Reject files larger than 1MB.
  - Reject outlines without `xmlUrl`.
  - Sanitize all string fields from the OPML (strip control characters, limit length).
  - Validate `xmlUrl` is a plausible HTTP(S) URL.

### Frontend

- **Import UI** — on the subscriptions page (`/podcasts/subscriptions`):
  - "Import OPML" button in the page header.
  - Opens a modal with:
    - File picker (accepts `.opml`, `.xml`).
    - "Import" button.
    - Progress/results view showing: total found, imported, skipped (already subscribed), errors.
    - Close button.
  - On success, refresh the subscriptions list.
- **Export UI** — on the subscriptions page:
  - "Export OPML" button in the page header (next to import).
  - Triggers download of the OPML file via `GET /podcasts/export/opml`.
  - No modal needed — direct download.

### BFF Layer

- `POST /api/podcasts/import/opml` — proxies multipart upload to backend.
- `GET /api/podcasts/export/opml` — proxies download.

## Acceptance

- uploading a valid OPML file with 10 podcasts creates subscriptions for all 10 and enqueues sync jobs.
- podcasts already subscribed are skipped (not errored) and reported in the `skipped_already_subscribed` count.
- podcasts not in PodcastIndex are still imported using OPML metadata (title + feed_url).
- feed URLs are normalized so `https://feed.example.com/rss` and `https://feed.example.com/rss/` don't create duplicates.
- OPML files with nested `<outline>` groups (common in Podcast Addict exports) are traversed recursively — all `type="rss"` outlines at any depth are found.
- files over 1MB or with more than 200 outlines are rejected with a clear error message.
- non-XML files are rejected with a clear error message.
- outlines without `xmlUrl` are skipped and counted in `skipped_invalid`.
- export returns a valid OPML 2.0 file that can be imported by Podcast Addict, Apple Podcasts, and other standard readers.
- export includes only `active` subscriptions (not unsubscribed).
- export file has correct Content-Type and Content-Disposition headers for browser download.
- import results modal shows a clear breakdown of what happened.
- after import, the subscriptions list reflects all newly subscribed podcasts.
- import is idempotent: importing the same file twice results in 0 new subscriptions on the second import.

## Non-Goals

- no import of episode-level state (played/unplayed, position) — OPML doesn't carry this.
- no import from non-OPML formats (JSON, CSV, etc.).
- no scheduled/automatic import (one-shot user action only).
- no import from a URL (file upload only).
- no feed fetching/validation during import beyond PodcastIndex lookup — feed reachability is verified during sync, not at import time.

## Key Decisions

**Feed URL as identity**: OPML podcasts are matched by `feed_url`, not by title or PodcastIndex ID. Feed URL is the universal identifier across podcast ecosystems. PodcastIndex lookup is a best-effort enrichment, not a requirement.

**Graceful degradation for unknown podcasts**: if PodcastIndex doesn't recognize a feed URL, we still create the podcast row from OPML metadata. The sync job will attempt to fetch the feed directly. This ensures import works even for niche/private feeds not indexed by PodcastIndex.

**200 outline limit**: prevents abuse and keeps the import request manageable. Most users have 20-100 podcast subscriptions. Power users with 200+ can do multiple imports.

**Synchronous parse, async sync**: the OPML parse + subscription creation happens synchronously in the request (fast — just DB inserts). The actual feed fetching and episode ingestion happens via background Celery jobs. This gives the user immediate feedback on what was imported.

## Risks

- malformed OPML files from various podcast apps. Podcast Addict, Apple Podcasts, Overcast, and Pocket Casts all produce slightly different OPML. Test with real exports from at least 3 apps.
- feed URLs in OPML may be outdated/dead. Sync jobs will fail for these, which is fine — the subscription still exists and the user can see the failure state.
- if a user imports 200 podcasts, that's 200 concurrent sync jobs. Consider throttling (e.g., enqueue in batches of 20 with delays) to avoid overwhelming the worker queue.
