# PR-03: Real Podcast Transcription Pipeline + Transcript Invariants

## Goal

Replace synthetic transcript-segment ingest with real podcast audio transcription work while preserving transcript stability and playback-fallback contracts.

## Builds On

PR-02 (Podcast Transcript UX + Subscription Lifecycle Closure) merged.

## Acceptance

- when subscribe/poll ingest encounters a newly created episode with a reachable external audio URL, transcription work is created and transcript segments come from transcription-provider output (not discovery/feed payload transcript fields).
- transcription work remains global and idempotent per episode media row: already-transcribed episodes are reused for subsequent subscribers without redundant provider transcription work.
- diarized transcription fallback is explicit: if diarized output fails, a non-diarized attempt is made and, when successful, transcript segments are persisted for normal reading/highlighting.
- transcript segment text is canonicalized for highlight/quote stability (NFC + whitespace normalization) before persistence.
- persisted transcript segments satisfy strict timing validity (`t_start_ms < t_end_ms`) and deterministic ordering guarantees for transcript rendering and query behavior; invalid segments are rejected (never coerced to zero-length).
- transcript failure semantics remain explicit and stable with `E_TRANSCRIPTION_FAILED`, `E_TRANSCRIPTION_TIMEOUT`, `E_DIARIZATION_FAILED`, and `E_TRANSCRIPT_UNAVAILABLE`, without regressing playback-only behavior for transcript-unavailable episodes.
- quote-to-chat from transcript highlights continues to include timestamp and speaker metadata when present, sourced from persisted transcription segments.
- existing podcast lifecycle guarantees remain intact: quota enforcement, episode identity/idempotency, active-subscription polling ingest, unsubscribe behavior, and visibility/processing-state regression suites stay green.

## Non-Goals

- no speaker-identity enrichment beyond diarization labels.
- no semantic/vector podcast search work.
- no advanced audio-player feature expansion beyond existing PR-02 UX.

## Key Decisions (if any beyond L2)

**Transcript Source Boundary**: discovery/feed metadata may describe episodes, but only transcription-provider output is authoritative for persisted transcript segments.

**Invariant Enforcement Layering**: transcript timing validity is enforced in ingestion validation and by database constraints so all write paths preserve `t_start_ms < t_end_ms`.

**Failure-Semantics Layering**: `media.last_error_code` represents terminal/readability state; `E_DIARIZATION_FAILED` is retained as transcription-attempt diagnostics when non-diarized fallback succeeds.

**Final-State Mapping**: diarization-fallback success yields readable media (`ready_for_reading`, no terminal transcript error), while true transcript failure outcomes map explicitly to `E_TRANSCRIPTION_FAILED`, `E_TRANSCRIPTION_TIMEOUT`, or `E_TRANSCRIPT_UNAVAILABLE` with playback-only behavior preserved for transcript-unavailable cases.

**Transcript Invariant Gate**: canonicalization and strict timing validity are enforced before transcript segments are accepted as readable artifacts.
