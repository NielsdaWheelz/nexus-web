# PR-11: MediaSession API + Player Streaming Error Handling

## Goal

Wire the Web MediaSession API so the player responds to hardware media keys, lock screen controls, spacebar play/pause, and headphone buttons. Also handle audio streaming errors (CORS, 404, network loss) gracefully instead of silently breaking.

## Builds On

PR-06 (Queue + Auto-Advance) merged. The player has full transport controls, queue, and auto-advance ‚Äî but none of it is accessible from outside the footer UI.

## Problem

1. **No media key support.** Pressing the play/pause key on a keyboard, headphone button, or lock screen does nothing. The browser's MediaSession API exists specifically for this ‚Äî it's ~50 lines of wiring but transforms the experience from "web page that plays audio" to "real audio app."
2. **No spacebar play/pause.** The most universal shortcut. Currently only arrow keys work, and only when the player footer is focused.
3. **No lock screen / notification center controls.** On mobile browsers that support MediaSession, the current track should appear in the OS notification area with play/pause/skip controls. Without MediaSession metadata, mobile users get a bare "playing audio" notification with no controls.
4. **No streaming error handling.** If the audio URL 404s, CORS-blocks, or the network drops mid-stream, the player silently stops. No error message, no recovery path. The user sees a frozen timecode with no explanation.

## Scope

### Frontend ‚Äî MediaSession

- **Register `navigator.mediaSession` metadata** ‚Äî on every `setTrack()` call, set:
  - `title`: episode title
  - `artist`: podcast name (from `track.podcast_title` ‚Äî extend `GlobalPlayerTrack` if needed)
  - `album`: podcast name (reuse)
  - `artwork`: podcast image URL (from `track.image_url` ‚Äî extend `GlobalPlayerTrack` if needed)
- **Register action handlers** on `navigator.mediaSession`:
  - `play` ‚Üí `play()`
  - `pause` ‚Üí `pause()`
  - `seekbackward` ‚Üí `skipBySeconds(-15)`
  - `seekforward` ‚Üí `skipBySeconds(30)`
  - `previoustrack` ‚Üí `playPreviousInQueue()`
  - `nexttrack` ‚Üí `playNextInQueue()`
  - `seekto` ‚Üí `seekToMs(details.seekTime * 1000)`
- **Update playback state** ‚Äî call `navigator.mediaSession.playbackState = 'playing' | 'paused' | 'none'` on play/pause/clear.
- **Update position state** ‚Äî call `navigator.mediaSession.setPositionState({ duration, playbackRate, position })` on timeupdate (throttled to every 1s to avoid excessive calls).
- **Cleanup** ‚Äî remove action handlers on `clearTrack()` and component unmount.
- **Feature detection** ‚Äî guard all calls behind `if ('mediaSession' in navigator)`. No-op on unsupported browsers.

### Frontend ‚Äî Global Keyboard Shortcuts

- **Spacebar play/pause** ‚Äî register a global `keydown` listener (on `document`) for Space key:
  - If a track is loaded: toggle play/pause.
  - If no track: no-op.
  - **Ignore** when focus is on an `<input>`, `<textarea>`, `<select>`, or `[contenteditable]` element (prevent conflicts with text editing).
- **Extend existing arrow key shortcuts** ‚Äî currently only work when the footer is focused. Move them to the global document listener alongside spacebar, with the same input-element guard.
- Register on mount of `GlobalPlayerProvider`, cleanup on unmount.

### Frontend ‚Äî Streaming Error Handling

- **Listen to `<audio>` error event** ‚Äî in `globalPlayer.tsx`, add an `error` event listener on the audio element.
- **Error state** ‚Äî add `playbackError: { code: number; message: string } | null` to the context.
  - Map `MediaError.code` to user messages:
    - `MEDIA_ERR_ABORTED` (1) ‚Üí "Playback was interrupted."
    - `MEDIA_ERR_NETWORK` (2) ‚Üí "Network error. Check your connection."
    - `MEDIA_ERR_DECODE` (3) ‚Üí "Audio format error."
    - `MEDIA_ERR_SRC_NOT_SUPPORTED` (4) ‚Üí "Audio URL unavailable."
  - Set `isPlaying = false` on error.
  - Clear error on next successful `play` or `setTrack()`.
- **Error UI in footer** ‚Äî when `playbackError` is set:
  - Replace the timecode area with the error message in a warning color.
  - Show a "Retry" button that calls `audio.load()` + `play()`.
  - Show an "Open source" link to `track.source_url` as a fallback.
- **Network recovery** ‚Äî listen to `window.addEventListener('online', ...)`. When the browser comes back online and `playbackError?.code === 2`, automatically retry playback.
- **Stalled/waiting states** ‚Äî listen to `<audio>` `waiting` and `stalled` events. Show a subtle loading indicator (spinner or pulsing timecode) during buffering. Clear on `playing` event.

### Extend GlobalPlayerTrack

Add optional fields to support MediaSession metadata:
- `podcast_title?: string` ‚Äî podcast name (for artist/album)
- `image_url?: string` ‚Äî podcast artwork (for lock screen)
- Callers (`setTrack()`) pass these from media detail or episode list data.

## Acceptance

- pressing spacebar toggles play/pause when a track is loaded; does nothing when no track; does not fire when typing in an input field.
- pressing keyboard media keys (play/pause, next, previous) controls the player.
- on mobile Safari/Chrome, lock screen shows track title, podcast name, artwork, and play/pause/skip controls via MediaSession.
- headphone button (single press) toggles play/pause.
- `navigator.mediaSession.setPositionState` updates seek position on lock screen.
- audio 404 shows "Audio URL unavailable" in the footer with a retry button and source link.
- network loss mid-stream shows "Network error" with automatic retry when connection recovers.
- buffering shows a loading indicator; clears when playback resumes.
- `playbackError` is cleared on successful play or track switch.
- all MediaSession code is guarded by feature detection ‚Äî no errors on unsupported browsers.
- arrow key shortcuts (left/right for skip) work globally (not just when footer focused), with input-element guard.

## Non-Goals

- no Picture-in-Picture (PiP) mode.
- no custom notification/toast for new episodes (separate feature).
- no service worker background audio (requires PWA setup).
- no retry queue for failed tracks (single retry button is sufficient).

## Key Decisions

**Global keyboard listeners on document**: simpler than managing focus state. The input-element guard (`activeElement.tagName`) prevents conflicts with all text inputs, textareas, selects, and contenteditable elements.

**MediaSession position updates throttled to 1s**: the spec recommends updating on timeupdate, but that fires ~4 times/second. 1s throttle is sufficient for lock screen scrubber accuracy.

**Automatic network recovery**: when `navigator.onLine` fires and the last error was network-related, auto-retry is safe because the user clearly wants to keep listening. No confirmation needed.

## Risks

- MediaSession behavior varies significantly across browsers and OS versions. Test on: Chrome desktop, Chrome Android, Safari iOS, Firefox desktop. Safari iOS has limited MediaSession support ‚Äî some action handlers may not fire.
- Spacebar global listener could conflict with page-level shortcuts in other parts of the app. The input-element guard mitigates this, but custom components with keyboard handlers (e.g., modals with Enter/Escape) should be audited.
- Audio error codes are coarse (only 4 values). A 403 and a 404 both map to `MEDIA_ERR_SRC_NOT_SUPPORTED`. The error message should be generic enough to cover both.
