# PR-02: Podcast Transcript UX + Subscription Lifecycle Closure

## Goal

Ship the user-facing podcast episode transcript experience and close subscription lifecycle behavior (unsubscribe + ongoing ingest) on top of PR-01 contracts.

## Builds On

PR-01 (Podcast Backend Foundation) merged.

## Acceptance

- transcript media detail responses expose an explicit playback-source contract for `podcast_episode`/`video`, so clients can render in-app playback and deterministic fallback behavior without guessing fields.
- users can open `podcast_episode` media in a transcript-first media pane with basic audio playback and deterministic segment ordering by timestamp.
- clicking a transcript segment seeks playback to that segment start time.
- if browser playback fails, the UI degrades to an explicit "open in source" action for the original audio URL while transcript reading/highlighting remain usable.
- transcript highlights remain anchored to transcript segment offsets, and quote-to-chat for transcript highlights includes timestamp plus speaker label when present.
- playback-only episodes (`processing_status=failed` with `E_TRANSCRIPT_UNAVAILABLE` and playable source) remain playable with explicit transcript-unavailable state; transcript-dependent actions stay disabled.
- unsubscribe supports all three constitution modes, defaults to mode 1, and never removes episodes from shared libraries implicitly.
- unsubscribe mode effects are deterministic across default-library and single-member-library retention/removal semantics, with no hidden cross-library deletions.
- active subscriptions ingest newly published episodes via polling, and this path reuses PR-01 idempotency + quota enforcement rules.
- visibility and processing-state regression suites remain green with added podcast transcript/player and lifecycle scenarios.

## Non-Goals

- no advanced player controls (speed, chapters, waveform, queue intelligence).
- no recommendation/ranking features or speaker-identity enrichment beyond provider diarization labels.
- no semantic/vector podcast search work.
- no audio hosting or proxy infrastructure.

## Key Decisions (if any beyond L2)

**Episode Entry Surface**: PR-02 ships episode UX through existing media/library flows; dedicated podcast discovery/subscribe browsing UI is deferred.

**Playback Source Contract**: transcript media responses use a typed playback-source surface (not inferred client behavior) sufficient for in-app playback and "open in source" fallback without local media hosting.

**Polling Eligibility Boundary**: only active subscriptions participate in ongoing polling ingest; unsubscribed records are excluded from future auto-add behavior.

## API Contract Notes

- `GET /media/{id}` and `GET /libraries/{id}/media` expose `playback_source` for transcript media (`podcast_episode`, `video`) as:
  - `kind`: `external_audio` | `external_video`
  - `stream_url`: playback target URL (audio element target for `external_audio`; provider watch/embed target for `external_video`)
  - `source_url`: URL for explicit "open in source" fallback
- `DELETE /podcasts/subscriptions/{podcast_id}` supports `mode=1|2|3` (default `1`) and returns subscription status including `unsubscribe_mode`.
- Polling ingest for active subscriptions reuses the same idempotent quota-aware sync path as direct subscription sync.
