# PR-01: Podcast Backend Foundation (Discovery, Subscription, Safe Ingest)

## Goal

Deliver a production-safe backend podcast vertical with global discovery, per-user subscriptions, bounded/idempotent episode ingest, and quota-governed transcription readiness.

## Builds On

Slice 6 merged state (media lifecycle/capabilities, default-library closure, highlights, quote-to-chat).

## Acceptance

- authenticated users can run podcast discovery globally and receive podcast metadata only; discovery results never include episode media rows.
- subscribing creates a per-user subscription and ingests only the newest plan-configured episode window into the subscriber’s default library.
- episode ingest is idempotent across retries, polling, and second-subscriber flows: GUID identity takes precedence and deterministic fallback identity prevents duplicates when GUID is absent.
- when an episode is already ingested globally, subsequent subscribers reuse the existing episode media row and receive library attachment without redundant transcription work.
- quota is enforced at new transcription-work creation: over-limit free-tier requests fail with a stable quota error and enqueue nothing; paid/manual plan changes apply immediately; usage resets at 00:00 UTC.
- transcription persists readable transcript-segment fragments with deterministic ordering guarantees (`(media_id, idx)` uniqueness and `(t_start_ms, idx)` display order), and diarization fallback persists non-diarized output.
- transcript-unavailable failures preserve playback-only behavior with explicit stable error-code semantics and consistent capability gating.
- existing visibility and processing-state regression suites remain green.

## Non-Goals

- no podcast transcript/player frontend UX in this PR.
- no unsubscribe mode implementation or subscription lifecycle UX closure.
- no semantic podcast search.
- no Stripe or self-serve billing integration.

## Key Decisions (if any beyond L2)

**Discovery Boundary**: podcast discovery remains a dedicated global discovery surface and does not reuse library-scoped content search semantics.

**Global Episode Identity Contract**: episode identity is enforced as a global, database-backed invariant (GUID first, deterministic fallback second) so all subscribers converge on shared episode rows.

**Quota Ledger Source of Truth**: transcription quota checks and daily usage accounting are persisted in primary transactional storage (UTC day buckets), not ephemeral runtime counters.

**Ingest vs Entitlement Split**: episode extraction/transcription is executed once per global episode row; user access is granted via library attachment through existing default-library provenance rules.
