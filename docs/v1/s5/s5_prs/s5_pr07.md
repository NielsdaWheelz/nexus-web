# pr-07: s5 hardening + slice 5 acceptance freeze

## goal
Close Slice 5 with explicit automated acceptance coverage, invariant/error audit traceability, and freeze-grade regression guardrails without expanding product scope.

## context
- PR-07 is the Slice 5 hardening/freeze PR and is explicitly hardening-only (`docs/v1/s5/s5_roadmap.md:153`).
- C8 ownership is slice-level acceptance closure and freeze gates for S5 sections 6-8 (`docs/v1/s5/s5_roadmap_ownership.md:10`, `docs/v1/s5/s5_roadmap_ownership.md:23`).
- S5 invariants and scenarios are normative for this PR (`docs/v1/s5/s5_spec.md:664`, `docs/v1/s5/s5_spec.md:688`, `docs/v1/s5/s5_spec.md:769`).
- S5 error-code usages audited by PR-07 are defined in the Slice 5 error table (`docs/v1/s5/s5_spec.md:638`).
- Most S5 EPUB coverage is already implemented across PR-02 through PR-06 tests, including chapter/TOC read APIs, retry, assets, highlights, and quote-to-chat compatibility (`python/tests/test_epub_ingest.py:213`, `python/tests/test_upload.py:870`, `python/tests/test_media.py:1255`, `python/tests/test_highlights.py:1637`, `python/tests/test_send_message.py:1316`, `apps/web/src/app/(authenticated)/conversations/page.test.tsx:91`).
- Current retry cleanup helper deletes persisted EPUB extraction artifacts (`epub_toc_nodes`, `fragment_blocks`, `fragments`) and does not currently target separate persisted chunk/embedding tables (`python/nexus/services/epub_lifecycle.py:308`).
- `embedding` is a readable lifecycle status in EPUB read/highlight paths, but no S5-owned backend embedding producer pipeline is present in this codebase (`python/nexus/services/epub_read.py:29`, `python/nexus/services/highlights.py:44`).
- Top-level roadmap reserves embedding implementation under S9 Semantic Search (`docs/v1/slice_roadmap.md:516`).

## dependencies
- PR-03 merged (EPUB ingest/retry lifecycle orchestration and retry cleanup semantics).
- PR-05 merged (EPUB reader baseline adoption, including `embedding` readability on UI path).
- PR-06 merged (highlight + quote-to-chat compatibility and attach handoff coverage).

### file: `python/tests/test_media.py`

**test: `test_epub_chapter_fragments_immutable_across_reads_and_highlight_churn`**
- input:
  - ready EPUB fixture with at least 3 chapter fragments and library linkage.
  - capture baseline fragment rows (`idx`, `html_sanitized`, `canonical_text`) from DB.
  - call `/media/{id}/chapters`, `/media/{id}/chapters/{idx}` repeatedly.
  - create and delete a highlight on chapter `idx=1` via existing highlight endpoints.
- output:
  - chapter reads and highlight operations succeed per existing contracts.
  - post-operation fragment rows match baseline bytes for every chapter (`html_sanitized`, `canonical_text` unchanged).

**test: `test_epub_fragment_content_stable_across_embedding_status_transition`**
- input:
  - ready EPUB fixture with chapter fragments and baseline DB snapshot of fragment content.
  - controlled status updates `ready_for_reading -> embedding -> ready` (no new embedding pipeline).
  - call `/media/{id}/chapters/{idx}` (and optionally `/chapters`) in `embedding` and `ready`.
- output:
  - read endpoints remain readable in `embedding` and `ready` per existing contract.
  - fragment DB content remains byte-for-byte equal to baseline across status changes.
  - no endpoint payload contract drift.

**test: `test_retry_epub_failed_clears_persisted_epub_artifacts_before_dispatch`**
- input:
  - failed EPUB fixture with valid source bytes and seeded extraction artifacts (`fragments`, `fragment_blocks`, `epub_toc_nodes`).
  - retry request by creator.
- output:
  - `202` with `processing_status='extracting'`, `retry_enqueued=true`.
  - dispatch invoked exactly once.
  - seeded extraction artifacts are absent after retry pre-dispatch reset transaction commits.
  - media row attempts increment and failure fields cleared.

**test: `test_retry_source_integrity_precondition_failure_no_mutation` (extend)**
- input:
  - failed EPUB fixture with seeded extraction artifacts and source-integrity precondition failure (missing/corrupt source object).
  - retry request by creator.
- output:
  - deterministic validation/storage error response (`E_STORAGE_MISSING` or existing mapped precondition code).
  - no dispatch.
  - artifact rows remain present and media lifecycle fields remain unchanged.

**test: `test_retry_preserves_source_identity_fields` (rerun as freeze evidence)**
- input:
  - failed EPUB fixture with valid source object and stored source identity fields (`media.file_sha256`, `media_file.storage_path`).
  - retry request by creator.
- output:
  - retry succeeds/follows existing contract.
  - source identity fields remain unchanged across retry reset/dispatch path.
