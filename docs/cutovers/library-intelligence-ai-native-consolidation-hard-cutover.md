# Library Intelligence — AI-Native Rebuild (Hard Cutover)

Status: SPEC — **Rev 3** (stable head + immutable revisions; not built)
Author: design synthesis, 2026-06-05
Type: hard cutover — greenfield, no production data, no fallbacks, no backward compatibility, no compat shims
Precedents: `0116` verifier drop ("AI-first chat keeps the model's answer as-is"); `current-only-artifacts-hard-cutover.md` (current-only for **source-derived** projections — citations, reader locators, source sets, default rendering; **generated LI work-product revisions are an explicit carveout**, see §0/§5.2); `library-entries-ownership-hard-cutover.md` (sole-writer + Tier-R, `EntryTarget`); `authors-directory-and-contributor-ownership-hard-cutover.md` (consolidate duplicates to one owner); `podcast-subsystem-ownership-hard-cutover.md` (one owner per state).

Citation unification is now at the **render contract**, not the storage table. Staleness keys on **content fingerprint**, not membership. Podcasts handled via `EntryTarget`. `run_kit` narrowed; `structured_synthesis` is a single call. Full disposition in §19.
>
> **Rev 3 changes:** the artifact becomes a **stable head** (`library_intelligence_artifacts`, one per library) + **immutable revisions** (`library_intelligence_artifact_revisions`) with `current_revision_id` promotion. Generated work-product history is retained; **current-only still governs source-derived projections** (citations, locators, source sets, default rendering) — generated revisions are the carveout. No `parent_revision_id`, no source-set/version identity. "Shadow + swap" → "draft revision + promote." Citations move from `artifact_id` to `revision_id`.

## 0. One-paragraph north star

Library Intelligence becomes a **first-class, source-grounded synthesis artifact that is also a chattable resource — like a document.** It has **one current published artifact per library** — a stable *head* whose generated content lives in **immutable revisions**, with the head promoting one revision to current (history retained; *current-only still governs source-derived projections — citations, reader locators, source sets, default rendering — see §5.2*). It is generated by an **Oracle-style structured LLM run** that reduces over **per-media "units"** (a reusable per-document summary + claims, each claim grounded to an existing evidence span), produces **synthesis prose with one-click-jump citations**, and persists that prose plus its own typed citations. "Chat on a library" is deleted and replaced by "chat on the artifact" through the **existing** `conversation_references` + library-scoped `app_search` machinery (no new linking work). The deterministic compiler and its nine tables are deleted. **No grand citation-table unification:** chat keeps `message_retrievals`, Oracle keeps folios, `conversation_references` stays. The shared thing is a **render contract** — one `CitationOut` read-model + `ReaderCitationData` adapter + the `[N]` jump — fed by three feature-specific typed stores. Chat and Oracle are touched only to adopt two behavior-preserving extractions (`run_kit`, `structured_synthesis`) and the shared render adapter.

## 1. Background — what exists today

### 1.1 Library Intelligence is a brain-empty scaffold
`services/library_intelligence.py:134` (`run_library_intelligence_build`) feeds `services/library_intelligence.py:860` (`_compile_sections`) — pure string templating, no LLM. `PROMPT_VERSION = "LibraryIntelligence.V1"` is a cache key with no prompt; `generator_model_id` is always `NULL`. The only "claim" is the tautology `'The library includes the source "<title>".'`; "Tensions" hardcodes `"No contradictions have been verified in the current deterministic pass."`. Frontend `LibraryIntelligenceView.tsx` renders this as if real. Backed by nine tables (`migrations/alembic/versions/0067_library_intelligence.py:21`).

### 1.2 The claim/evidence/support-state taxonomy was already deleted once
`0066` created chat's verifier stack with a `support_status` enum byte-identical to `library_intelligence_claims.support_state`. `0116` (2026-05-27) hard-dropped chat's stack — irreversible, head-asserted — under *"AI-first chat keeps the model's answer as-is."* `library_intelligence`'s `0067` copy is the orphaned survivor. This cutover finishes that decision.

### 1.3 What we reuse (verified)
- **Chattable resources:** `ConversationReference(conversation_id, resource_uri)` + `resource_resolver.ResourceUriScheme` (already includes `media`, `library`) + `context_assembler._build_resources_block`. `LibraryChatTab.tsx` already does `initial_references:["library:uuid"]`.
- **Library-scoped retrieval:** `app_search` accepts `scopes:["library:uuid"]`, validated against conversation references, filtered via `library_entries` join (hybrid `content_embeddings` pgvector-256 + `ts_rank_cd`).
- **Per-media content substrate:** `content_chunks` (ordered by `chunk_idx`), `evidence_spans` (durable citeable spans, `selector`, `resolver_kind`), `content_embeddings`. Citation jump: `hrefForReaderTarget()` → `#evidence-/#fragment-/#page-/#t-`, resolved by `locator_resolver` against the **active** index run.
- **Durable run + SSE:** `api/routes/_sse.py` (`format_sse_event`, `tail_cursor_stream`, `open_sse_listener` over `LISTEN/NOTIFY`) — already shared; the event-append/finalize/worker-boundary are duplicated (chat `chat_run_event_store.append_run_event` auto-seq vs Oracle `oracle._append_event` manual seq).

### 1.4 What we deliberately do NOT touch (review findings 3, 6)
- **`message_retrievals` stays.** FK from `message_retrieval_candidate_ledgers.retrieval_id`; reconstructed for message replay on every GET (`chat_run_message_blocks._retrieval_result_blocks_for_message`); backs both app + web search; `included_in_prompt` prompt-tracking; cleanup cascades. Not an edge — a retrieval event-record.
- **`conversation_references` stays.** 12 consumer systems: initial refs, `is_conversation_reference` admission gates (`read_resource`, `inspect_resource`, `chat_run_validation`), `app_search` scope validation, context assembly, `reference_added` SSE, reverse highlight→conversation lookup, search scoping. A conversation-boundary abstraction over opaque URIs, not a workspace relationship.
- **`oracle_reading_passages` stays.** Carries `phase` (descent/ordeal/ascent) + `marginalia_text` (per-passage generated commentary) + `attribution_text`; `compute_concordance` cross-joins it. A citation **+ generated content**, not a pure citation.
- **`object_links` stays as-is.** Symmetric for unlocated pairs (LEAST/GREATEST unique index), user-facing per-call-commit CRUD, lacks a `library` type. Wrong home for directed machine citations.

## 2. Goals

G1. **Real intelligence.** LLM-generated, source-grounded synthesis with click-to-passage citations. Match NotebookLM's "verifiability over coverage"; exceed it on whole-library scope + cross-source tensions + open questions.
G2. **One owner per *primitive that is genuinely shared*:** durable-run/event-stream, structured synthesis, the per-media unit, and the **citation render contract** (read-model + adapter + jump). Storage stays feature-typed where the data is not homogeneous (findings 3, 6).
G3. **AI-first, minimal model.** Nine LI tables → three small ones. No verifier. Structure only where reused (units); prose for the synthesis.
G4. **Stable head + immutable revisions.** One current artifact *head* per library; generate/regenerate/update creates a new **immutable revision**; success atomically **promotes** that revision to current (`current_revision_id`). No source-set versions, no parent-revision lineage, no source-replay guarantee. Current-only still governs source-derived projections (see §5.2).
G5. **Reusable per-media units.** A summary + claims service usable on ingest, on-demand, in `app_search` cards, the reader, and the library list.
G6. **House rules.** Sole-writer + Tier-R; typed contracts (no `dict[str, Any]` across boundaries); `SERIALIZABLE` + bounded retry; push SSE (no polling); no `ON DELETE CASCADE`; exhaustive matching; make-illegal-states-unrepresentable.

## 3. Non-goals

N1. **No post-hoc verifier** (`0116`). Grounding-by-construction + one model-emitted inline role; justification in prose, not a `support_state` column.
N2. **No Rev 2 storage fold.** The citation/context/retrieval/folio stores (`message_retrievals`, `conversation_references`, `oracle_reading_passages`, `object_links`) remain feature-typed in this cutover (findings 3, 6) — **until a later provenance-graph cutover (§20) can preserve their distinct invariants without compatibility shims.** Separation here is a safety decision, not the end state. Chat and Oracle keep their citation stores untouched.
N3. **No source-derived version identity, no source-set identity, no LI fork/DAG model, no `parent_revision_id`, no historical citation replay.** Revision history exists only for generated LI work-products (the synthesis prose + its citations), never for source projections.
N4. **No auto-rebuild.** Staleness surfaced; generation user-triggered.
N5. **No audio/video/Anki overview, no ambient/proactive surfacing, no bi-temporal graph.** Foundations only.
N6. **No sources panel** for uncited retrievals.
N7. **No streamed token synthesis required.** The reduce is a single structured call (like Oracle); prose appears on completion. (Token-streaming is an optional later enhancement, not in scope.)

## 4. Target behavior (user-facing)

1. Open **Intelligence** on a library. No artifact → empty state + **Generate**. Artifact exists → the **synthesis prose** (overview, key topics, key sources, reading path, tensions, open questions — prose, not typed sections) with `[N]` citations that **jump to the exact source passage**.
2. **Status line:** `Current` / `Stale — N sources changed — Regenerate?` / `Generating…` (streamed progress) / `Failed — Retry`. One status; no stat cards; no build-state split.
3. **Regenerate/update** = explicit. The current artifact **stays visible** while a **draft revision** builds; on success the draft is **promoted to current**; previous revisions are **retained** (immutable). Default UI shows **current only**; an optional lightweight history/restore is allowed — no branch graph.
4. **Chat on it.** A "Chat" affordance opens a normal conversation with the **artifact + library** referenced (existing `conversation_references`), retrieval **scoped to the library** (existing `app_search` scope). No separate "library chat."
5. Streaming is **push** (LISTEN/NOTIFY → SSE).

## 5. Architecture — final state

### 5.1 Composition

```
 shared primitives (one owner each)
   run_kit:            seq + append(auto-seq) + SSE tail + terminal-transition + worker-boundary   (generic ONLY)
   structured_synthesis: render prompt + JSON contract → ONE structured call → validated T → emit events
   citation render contract: CitationOut (backend read-model) + ReaderCitationData adapter (frontend) + [N] jump
   media_intelligence:  per-media summary + claims (grounded to existing evidence_spans)

 features compose subsets — storage stays feature-typed:
   chat    → run_kit ; message_retrievals (UNCHANGED) → CitationOut ; native tool-loop ; domain finalize stays in chat
   oracle  → run_kit ; structured_synthesis ; oracle_reading_passages (UNCHANGED) ; concordance/marginalia stay
   library_intelligence → run_kit ; structured_synthesis ; per-media units → reduce → prose + library_intelligence_citations
                          ; chattable via conversation_references + library app_search scope
```

- **`run_kit`** owns *only* the generic run mechanics: monotonic `seq`, append (auto-seq, the chat pattern), SSE tail, the terminal status transition, and the worker-boundary exception finalizer. **Domain finalization stays per-feature** — chat writes assistant content/message-doc/usage; Oracle mutates reading state + emits domain events; each calls `run_kit` only to mark terminal + emit `done`. (Finding 5.)
- **`structured_synthesis`** is a **single** structured call (matching Oracle today, which is *not* token-streamed): render prompt + JSON schema → one `llm.generate` → validated typed object → emit progress events. Oracle's `_build_llm_request`/`_parse_llm_output` become callers; the LI reduce is a caller. (Finding 5.)
- **Citation unification = render contract only** (findings 3, 6): a backend `CitationOut` read-model and a frontend `ReaderCitationData` adapter + the existing jump, fed by three **untouched/feature-typed** stores (`message_retrievals`, `oracle_reading_passages`, new `library_intelligence_citations`).
- **`media_intelligence`** is the shared per-media unit owner; **`library_intelligence`** is the slim artifact owner.

(Nothing else dropped — finding 9 teardown is now small; see §11.)

### 5.3 Citation render contract (the actual unification)

- **Backend `CitationOut`** (shared read-model): `{ordinal, role, target_ref:(type,id), locator, deep_link, snapshot}`. Produced by: chat (from the cited subset of `message_retrievals`), Oracle (from folios, for the jump only), Library Intelligence (from `library_intelligence_citations`). Storage stays feature-typed; the **read-model** is shared.
- **Frontend `ReaderCitationData` adapter** (finding 7): one `CitationOut → ReaderCitationData` function + locator normalization. `buildCitations` is generalized to consume `CitationOut[]` instead of being `message_retrievals`-shaped. `MarkdownMessage`/`ReaderCitation`/`hrefForReaderTarget` unchanged; the `[N]` jump is reused by all three features.
- **Render is one-level:** the citation points at a resolvable node (`evidence_span`/`content_chunk`/`media`); the jump resolves against the **active** index run.
- **Provenance is two-level by construction:** the reduce may only cite the `media_claims` it was given; each carries a `not-null evidence_span_id`; the artifact citation inherits that span. The model **cannot cite a span that does not exist** (no verifier needed).

### 5.4 Staleness (finding 2 — content-fingerprint, not membership)

Head `status` computed at read:
- `unavailable` if no head or no `current_revision_id`.
- `building`/`failed` from the in-flight **draft revision**'s run (a draft can build while the current revision stays shown).
- else resolve the current `library_entries` for the library to its **media set** (media → itself; podcast → its episodes' media), with each media's current `media_summaries.content_fingerprint`. **`stale`** if that `{media_id → fingerprint}` map ≠ the **current revision's** `covered_targets` snapshot (membership change **or** any covered source re-ingested); else **`current`**.
- Rationale: `locator_resolver` fails closed when a citation's `index_run_id` ≠ the media's active run, so a re-ingested source silently kills that source's `[N]` jumps unless we stale on content change. Cheap: the unit `content_fingerprint` is the signal.

## 6. Capability contracts (typed)

### 6.1 `services/run_kit` (new — generic only)
```
append_event(db, *, stream: RunStream, event_type: RunEventType, payload: RunEventPayload) -> int   # auto-seq
mark_terminal(db, *, stream: RunStream, status: TerminalStatus) -> None                              # transition + 'done' event ONLY
fail_after_worker_exception(db, *, stream: RunStream, error: BaseException) -> None
tail(stream: RunStream) -> SSE                                                                       # reuses _sse.tail_cursor_stream
```
- `RunStream` discriminates `chat_run | oracle_reading | library_intelligence` (event table + parent col internal). **Does not** write domain content/usage/state — callers do that, then `mark_terminal`. Replaces the auto-seq/manual-seq duplication; one LISTEN/NOTIFY channel per kind.

### 6.2 `services/structured_synthesis` (new, from Oracle)
```
run_structured_synthesis(*, llm, request: SynthesisRequest, schema: type[T],
                         on_event: Callable[[RunEventPayload], None]) -> T   # ONE call, validated, events emitted
```

### 6.3 Citation read-model + adapter
```
# backend
build_citation_outs_for_message(db, *, assistant_message_id) -> list[CitationOut]   # from message_retrievals (cited subset)
build_citation_outs_for_revision(db, *, revision_id) -> list[CitationOut]           # from library_intelligence_citations
# frontend
toReaderCitationData(c: CitationOut): ReaderCitationData                            # + locator normalization
```

### 6.4 `services/media_intelligence` (new, shared — sole writer of `media_summaries`/`media_claims`)
```
ensure_media_unit(db, *, media_id) -> MediaUnitRef     # idempotent on content_fingerprint; enqueue build if missing/stale
get_media_unit(db, *, media_id) -> MediaUnit | NotReady
run_media_unit_build(db, *, media_id) -> None          # worker: structured_synthesis over content_chunks → summary + claims(+evidence_span_id)
```
- Trigger: enqueued after content indexing completes; on-demand `POST /media/{id}/summarize`. Feeds the reduce, `app_search` result cards, reader, library list.

### 6.5 `services/library_intelligence` (rewritten, slim — sole writer of the head, revisions, + `library_intelligence_citations`)
```
get_artifact(db, *, viewer_id, library_id) -> ArtifactView          # head + current-revision content + computed status (§5.4)
generate_artifact(db, *, viewer_id, library_id, request_token) -> RevisionRef   # 202; creates a draft revision; request-token idempotent
run_artifact_generation(db, *, revision_id) -> None                 # worker: resolve targets→media, ensure_media_unit each,
                                                                    # structured_synthesis reduce over units → prose + citations,
                                                                    # write covered_targets, then PROMOTE (below)
list_revisions(db, *, viewer_id, library_id) -> list[RevisionSummary]            # optional history (route deferrable)
promote_revision(db, *, viewer_id, revision_id) -> None                          # optional restore (route deferrable)
```
- **Draft revision → promote** (no shadow head, no `unique(library_id)` collision): (1) find-or-create the stable **head** row; (2) create a `building` **revision**; (3) run synthesis into that revision; (4) write the revision's citations; (5) in one `SERIALIZABLE` tx, mark the revision `ready`, set `promoted_at`, and set `head.current_revision_id = revision.id`. A failed revision stays `failed`; the **current revision is unchanged**. Previous revisions are retained (immutable), addressable for optional restore.

### 6.6 Chattable artifact (reuses `conversation_references` — no new linking)
- Add `library_intelligence_artifact` to `resource_resolver.ResourceUriScheme` + a `_present()` handler (inline synthesis if under threshold, else label+summary+`fetch_hint`) + a resource loader. `library` scheme already exists.
- "Chat on the artifact" = `POST /conversations {initial_references:["library_intelligence_artifact:uuid","library:uuid"]}` → existing `conversation_references` rows → `app_search` scoped to the library. **No object_links, no schema change to references.**
- The reference is to the **head** (`library_intelligence_artifact:<artifact_id>`); it **resolves to the current revision at prompt-assembly time**. The chat run records the **`revision_id` it actually used** in run metadata (reproducibility without a source-replay guarantee).
- If a chat action generates an artifact update, it creates a **new revision** through `library_intelligence`; the user explicitly promotes it (or the update action promotes on success). **Do not revive `artifact_intent` or `message_artifacts`.**

## 7. API design

| Method | Route | Handler → service | Notes |
|---|---|---|---|
| GET | `/libraries/{id}/intelligence` | `get_artifact` | `{artifact_id, revision_id, status, content_md, citations, build}` (current revision); **no** version/source-set fields |
| POST | `/libraries/{id}/intelligence/generate` | `generate_artifact` | 202 `{artifact_id, revision_id, run_id}`; **idempotency = client `request_token`**, so a no-change regenerate still builds a fresh draft revision |
| GET (SSE) | `/stream/library-intelligence/{revision_id}/events` | `run_kit.tail` | reuses `_sse`; streams the draft revision's build |
| GET | `/libraries/{id}/intelligence/revisions` | `list_revisions` | **optional** history; defer if not shipping restore now |
| POST | `/libraries/{id}/intelligence/revisions/{revision_id}/promote` | `promote_revision` | **optional** restore; defer if not shipping now |
| POST | `/media/{id}/summarize` | `ensure_media_unit` | also auto-enqueued at ingest |
| POST | `/conversations` | existing `conversations.create` | `initial_references:["library_intelligence_artifact:uuid","library:uuid"]` |

GET returns the **current revision's** ready content + an optional **in-flight build** block (finding 8): `{artifact_id, revision_id, status, content_md, citations, build:<draft-revision run status or null>}`, so the UI shows the current revision while a draft regenerates. **Note on history routes:** if `/revisions` + `/promote` are deferred, retained revisions are audit-only — without a restore path they carry little product value, so ship at least `promote` to make history a feature.

Chat/Oracle citation read paths are **unchanged on the wire**; only their *frontend* citation construction routes through the shared `toReaderCitationData` adapter.

## 8. Frontend design

- **Delete** `LibraryIntelligenceView.tsx` deterministic UI. New pane renders the current revision's `content_md` via **`MarkdownMessage`** (reused: `[N]` → `CitationAwareLink` → `ReaderCitation` + jump). Citations come from `build_citation_outs_for_revision(current_revision_id)` → `toReaderCitationData`.
- **Status + Generate/Regenerate**; while a draft is `building`, subscribe to the draft revision's SSE (replace `useResource` polling). Default UI shows the current revision only; an **optional** lightweight history/restore list is allowed (no branch graph).
- **Generalize `buildCitations`** to a `CitationOut[]` input + the `toReaderCitationData` adapter (finding 7); chat is refactored to feed `CitationOut` (no storage change). `readerTarget`/`ReaderCitation`/jump unchanged.
- **Delete** the `library-chat` surface (`LibraryChatTab.tsx`, the `library-chat` entry in `paneSecondaryModel.ts`); the pane's "Chat" action drives chat-on-artifact.
- `app_search` result cards gain per-media `summary_md`.

## 9. Composition with existing systems

- **Chat:** adopts `run_kit` (behavior-preserving) + the `CitationOut` adapter (frontend only). `message_retrievals` untouched. May optionally accept library-scoped retrieval when chatting on an artifact (existing scope path).
- **Oracle:** adopts `run_kit` + `structured_synthesis` (behavior-preserving). Folios, concordance, marginalia untouched; can also emit `CitationOut` for jumps.
- **Reader/Ingest/Search:** consume per-media `summary_md`; ingest enqueues unit builds; jump machinery unchanged.

## 13. Acceptance criteria

- **AC-1 (real synthesis).** Generation over a non-empty library produces LLM-derived prose, ≥1 citation per substantive claim; zero tautological "library includes the source X."
- **AC-2 (grounding-by-construction).** Every `library_intelligence_citations` row targets an existing resolvable node; a `media_claim` with no `evidence_span_id` is not emitted; the reduce can only cite provided unit-claims (tested).
- **AC-3 (one-click jump).** `[N]` in the artifact and in a library-scoped chat navigate to the exact passage via the unchanged jump.
- **AC-4 (chattable, unified).** "Chat on the artifact" opens a conversation referencing artifact + library, retrieval scoped to the library; no `library-chat` surface/route remains.
- **AC-5 (head + revisions).** Exactly one stable artifact **head** per library; generate/regenerate/update creates **immutable revisions**; the current head swaps atomically by setting `current_revision_id`; old revisions remain addressable; **no `parent_revision_id`**, no source-set/version identity.
- **AC-6 (content staleness).** Adding/removing a source **or re-ingesting an existing source** flips status to `stale` with the changed count; rebuild is user-triggered only. (Reverses old membership-only.)
- **AC-7 (podcasts).** A library with podcast entries covers their episode-media in units and coverage; a new episode flips `stale`.
- **AC-8 (shared primitives).** Chat, Oracle, LI runs stream via `run_kit`/`_sse`; Oracle + reduce call `structured_synthesis`; all three render citations via `toReaderCitationData`. (import-graph + grep gates.)
- **AC-9 (units reused).** A unit built once is reused by reduce, `app_search` cards, reader; `/media/{id}/summarize` returns the same unit.
- **AC-10 (push not poll).** Artifact pane consumes SSE; no `useResource` polling of the intelligence endpoint.
- **AC-11 (untouched stores intact).** `message_retrievals`, `conversation_references`, `oracle_reading_passages`, `object_links` exist and their tests pass unchanged.
- **AC-12 (no premature fold).** This cutover passes only if those four stores still exist with their consumers intact (§14 must-REMAIN gates). A later provenance-graph cutover may supersede this separation only by independently proving — with no compatibility shims — that **retrieval replay, scope admission, concordance/marginalia, and user-link CRUD** all survive (§20.5).

## 15. Rules & invariants

- **Sole writers:** the head, revisions, + `library_intelligence_citations` ← `library_intelligence.py`; `media_summaries`/`media_claims` ← `media_intelligence.py`.
- **Tier-R reads** via the owning service's read model; visibility in `auth/permissions.py`.
- **Concurrency:** promotion runs `SERIALIZABLE` (mark revision `ready` + set `head.current_revision_id` atomically); revisions are append-only so concurrent drafts never collide (no `unique(library_id)` shadow contention); unit builds idempotent on `content_fingerprint`; job enqueue + row insert atomic with `pg_notify`.
- **Exhaustiveness:** `status`, `role`, `RunStream`, `RunEventType`, `EntryTarget` are discriminated unions with `assert_never`.
- **Defects vs errors:** a cited span missing at *generation* is impossible (claims carry not-null spans); a cited span unresolvable at *read* (post re-ingest) is the `stale` signal, not a defect.
- **Consolidate by invariant, not by table shape** (§20.2): the only sanctioned axes are authorization boundary, retrieval replay, generated content, user relationship, or citation render. This rule is what retroactively explains the Rev 1 reversal — and what governs every future fold.

## 20. Future Direction: Resource Provenance Graph

Long-term consolidation of these stores **remains desirable** — Rev 2 keeps them separate for safety, not because separation is the end state. But the right consolidation is **not** a fold into today's `object_links` (symmetric, user-CRUD, undirected, opaque-URI-adjacent). The future primitive is a **typed `ResourceRef` plus a single provenance-edge owner**: a directed, role-bearing edge graph whose one owner preserves each current store's invariant as a distinct edge kind — never by flattening them into a shared row shape.

### 20.1 Invariant taxonomy (what each store actually IS)

| Store | Invariant kind | What must be preserved |
|---|---|---|
| `conversation_references` | **ContextRef** | authorization / scope-admission boundary (12 consumers gate on it) |
| `message_retrievals` | **RetrievalLedger + CitationEdge** | retrieval replay (reconstructed on every message GET) **+** the cited subset |
| `oracle_reading_passages` | **GeneratedFolio + CitationEdge** | generated per-passage content (phase, marginalia, attribution) + concordance **+** the cited subset |
| `object_links` | **WorkspaceRelation** | user-facing relationship CRUD (symmetric, user-owned) |
| `library_intelligence_citations` | **CitationEdge** | the artifact's cited subset (the one clean new edge) |

Three stores share a **CitationEdge** invariant. That edge — and *only* that edge — is what Rev 2 unifies, and only at the **render contract** (`CitationOut`/`ReaderCitationData`), never in storage.

### 20.2 SME rule (governs all future consolidation)

> **Do not consolidate by table shape. Consolidate only by invariant** — one of: **authorization boundary, retrieval replay, generated content, user relationship, or citation render.** Two tables sharing `(a_id, b_id, role)` columns are not foldable if they carry different invariants; two tables with different columns may legitimately share one invariant (the CitationEdge spread across `message_retrievals` / folios / `library_intelligence_citations`). Shape-driven folds are the Rev 1 mistake.

### 20.3 Proving ground (dependency order before any storage fold)

Storage consolidation is blocked until the **canonical contracts** are landed and hardened as the proving ground:
- **`ResourceRef`** — the typed, directed reference that supersedes opaque `resource_uri` strings,
- **`CitationOut`** — the backend citation read-model,
- **`ReaderCitationData`** — the frontend render shape,
- **locator normalization** — one jump contract across features.

If those four hold across chat, Oracle, and LI with no per-feature special-casing, a provenance-graph owner becomes tractable. Until then, storage stays feature-typed. (Rev 2 lands the last three; `ResourceRef` is the remaining prerequisite.)

### 20.4 `object_links` must not become the graph owner by accretion

`object_links` should end one of exactly two ways: **stay** the user-facing workspace-relations table (its current invariant), or **become one edge kind inside** a new provenance-graph owner. It must **not** silently accrete machine-derived citation/context/retrieval edges, directionality, and new types until it is the de-facto graph owner that was never designed as one. Any rename/narrowing is a deliberate, separate cutover — not drift.

### 20.5 The future cutover's burden of proof

A provenance-graph cutover supersedes Rev 2's "keep separate" **only if it proves, with no compatibility shims, that all four load-bearing invariants survive**: **retrieval replay** (message reconstruction on GET), **scope admission** (the 12 `conversation_references` gates), **concordance + marginalia** (Oracle folios), and **user-link CRUD** (`object_links`). Failing any one means the fold is premature — precisely the Rev 1 error this section exists to prevent.
